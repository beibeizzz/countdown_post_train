# V2 verl GRPO Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Train the Full SFT actor with stock verl 0.6.0 GRPO on two GPUs using FSDP2 and vLLM, exact Countdown rewards, fixed-50 native validation, resumable checkpoints, and post-training best/final Hugging Face exports.

**Architecture:** V2 owns only data conversion, reward functions, configuration composition, launch wrappers, validation selection, export orchestration, and metrics post-processing. It imports installed verl APIs and never patches or copies verl source. Native verl checkpoints remain the continuation authority; Hugging Face exports are derived after training with `verl.model_merger`.

**Tech Stack:** verl 0.6.0, Ray 2.48.0, FSDP2, vLLM 0.9.1, PyArrow 19.0.1, Transformers 4.53.2, W&B, pytest.

- [ ] **Step 2: Implement**

Aggregate only serializable numbers. Preserve zero-standard-deviation groups;

- [ ] **Step 3: Connect the metrics sidecar**

The launcher creates one W&B run ID and exports it to both verl and the
sidecar. The sidecar watches `trainer.rollout_data_dir` for atomic
step-addressed JSONL files, computes the project metrics once per trainer
step, appends `post_train_v2/outputs/grpo/run/metrics.jsonl`, and logs with
the same W&B run ID and numeric step. It must not create a second run name.
Stock verl remains responsible for policy loss, learning rate, native reward,
length, and throughput; the sidecar adds `reward_std`, `group_reward_std`,
`frac_reward_zero_std`, all-correct/all-wrong fractions, format rate,
accuracy, truncation count, and bucket metrics.
