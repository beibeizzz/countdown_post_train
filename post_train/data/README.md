# Data

Pipeline data products are written here. Files under this directory are generated by scripts and can be regenerated from raw inputs and configs.

Subdirectories:

- `processed`: solver-backed source, validation, eval subset, and test files.
- `teacher_rollouts`: teacher accepted/rejected generations.
- `sft`: SFT/RFT training data.
- `dpo`: DPO pair data and candidate logs.
- `grpo`: GRPO prompt data.
- `eval`: standalone evaluation outputs.

All generated `manifest.json` files use the shared `countdown.post_train.manifest.v1` envelope and add stage-specific counts/settings.
